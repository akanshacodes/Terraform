# 5. Imperative vs Declarative Infrastructure

One of the most important DevOps interview topics.

---

# Imperative Approach

Step-by-step instructions are provided to the system.

---

## Example

```text
Create Resource Group
Create Storage Account
Attach Networking
```

---

# Characteristics

- Command-based
- Procedural execution
- Explicit steps

---

# Declarative Approach

Desired final state is defined.

---

## Example

```text
Deploy complete infrastructure
```

---

# Characteristics

- Desired-state model
- Tool determines execution steps
- Easier scalability

---

# Tool Mapping

| Approach | Tool |
|---|---|
| Imperative | Azure CLI |
| Declarative | Terraform |

---

# Important Interview Point

```text
Terraform is declarative, while Azure CLI is imperative.
```

---

# 8. Terraform Fundamentals

|---|---|

---

```hcl
}
```

---

```hcl
}
```

```hcl
}
```

```hcl
}
```

# 20. Resource Block Deep Dive

Resource blocks define infrastructure objects.

---

# Example

```hcl

  name     = "rg-prod"
  location = "Central India"
}
```

---

# Breakdown

| Component | Purpose |
|---|---|
| resource | Resource declaration |
| azurerm_resource_group | Azure RG type |
| example1 | Local Terraform name |
| name | Actual Azure RG name |
| location | Azure region |
