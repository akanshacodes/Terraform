# 1️⃣ What happens when you run `terraform init`?

## ✅ Answer

`terraform init` initializes the Terraform working directory.

It performs the following tasks:

- Initializes backend configuration
- Downloads required provider plugins
- Creates `.terraform` folder
- Creates `.terraform.lock.hcl`
- Prepares Terraform for execution

---

## 📌 Example Output

```bash
Initializing the backend...
Initializing provider plugins...
- Installing hashicorp/azurerm...
Terraform has been successfully initialized!
```

---

# 2️⃣ Why is `terraform init` important?

## ✅ Answer

Without running `terraform init`, Terraform cannot:

- Download providers
- Configure backend
- Execute Terraform properly

Commands like `terraform plan` or `terraform apply` may fail.

---

# 3️⃣ When should you run `terraform init`?

## ✅ Answer

Run `terraform init` when:

- Starting a new Terraform project
- Cloning Terraform code from GitHub
- Changing provider versions
- Changing backend configuration

---

# 4️⃣ What is the `.terraform` folder?

## ✅ Answer

`.terraform` is an automatically generated directory created after `terraform init`.

It stores:

- Provider plugins
- Module cache
- Backend metadata

---

## 📌 When is it created?

```bash
terraform init
```

After running this command.

---

# 5️⃣ What is `.terraform.lock.hcl`?

## ✅ Answer

`.terraform.lock.hcl` is the provider lock file.

It stores:

- Exact provider versions
- Checksums/signatures

Purpose:

- Ensures version consistency
- Prevents unexpected provider upgrades

---

## 📌 Example

```hcl
provider "azurerm" {
  version = "4.73.0"
}
```

---

# 6️⃣ Should `.terraform.lock.hcl` be committed to Git?

## ✅ Answer

Yes.

It should be committed because it ensures all team members use the same provider versions.

---

# 7️⃣ What does `terraform validate` do?

## ✅ Answer

`terraform validate` checks whether Terraform configuration syntax is valid.

It checks:

- Syntax errors
- Invalid arguments
- Incorrect blocks

It does NOT:

- Create resources
- Connect to Azure
- Deploy infrastructure

---

## 📌 Example

```bash
terraform validate
```

Output:

```bash
Success! The configuration is valid.
```

---

# 8️⃣ Difference between `terraform validate` and `terraform plan`

| terraform validate | terraform plan |
|---|---|
| Checks syntax only | Shows infrastructure changes |
| No Azure API calls | Connects to Azure |
| Faster | Slower |
| No execution plan | Generates execution plan |

---

# 9️⃣ What does `terraform fmt` do?

## ✅ Answer

`terraform fmt` formats Terraform code automatically.

Benefits:

- Better readability
- Standard formatting
- Clean indentation

---

## 📌 Example

### Before

```hcl
name="test-rg"
```

### After

```hcl
name = "test-rg"
```

---

# 🔟 What does `terraform plan` do?

## ✅ Answer

`terraform plan` shows what changes Terraform will make before deployment.

It compares:

- Desired configuration
- Current infrastructure state

---

## 📌 Example

```bash
Plan: 1 to add, 0 to change, 0 to destroy.
```

Meaning:

- 1 resource will be created
- No modifications
- No deletions

---

# 1️⃣1️⃣ Why is `terraform plan` important in production?

## ✅ Answer

It helps avoid accidental infrastructure changes.

Benefits:

- Review changes before deployment
- Detect mistakes
- Prevent downtime
- Validate infrastructure safely

---

# 1️⃣2️⃣ What does `terraform apply` do?

## ✅ Answer

`terraform apply` executes infrastructure changes.

It:

- Creates resources
- Updates resources
- Deletes resources

---

## 📌 Example

```bash
Apply complete! Resources: 1 added, 0 changed, 0 destroyed.
```

---

# 1️⃣3️⃣ What happens internally during `terraform apply`?

## ✅ Answer

Terraform performs these steps:

1. Reads Terraform configuration
2. Reads state file
3. Generates execution plan
4. Requests approval
5. Calls Azure APIs
6. Creates infrastructure
7. Updates state file

---

# 1️⃣4️⃣ What is `terraform apply -auto-approve`?

## ✅ Answer

This command skips manual approval.

---

## 📌 Example

```bash
terraform apply -auto-approve
```

---

## 📌 Mostly used in:

- CI/CD pipelines
- Automation scripts

---

## ⚠️ Risk

It can deploy unintended changes without confirmation.

---

# 1️⃣5️⃣ What is `terraform.tfstate`?

## ✅ Answer

`terraform.tfstate` is the Terraform state file.

Terraform uses it to track:

- Created resources
- Resource IDs
- Infrastructure mapping

---

# 1️⃣6️⃣ When is `terraform.tfstate` created?

## ✅ Answer

It gets created after:

```bash
terraform apply
```

Once infrastructure deployment succeeds.

---

# 1️⃣7️⃣ What is `terraform.tfstate.backup`?

## ✅ Answer

Terraform automatically creates a backup of the previous state file.

Purpose:

- Recovery
- Rollback support

---

# 1️⃣8️⃣ When is `terraform.tfstate.backup` created?

## ✅ Answer

It is created when:

- Existing state changes
- New `terraform apply` runs

Terraform first backs up old state, then updates new state.

# 2️⃣0️⃣ Explain this provider block.

```hcl
provider "azurerm" {
  features {}
  subscription_id = "xxxx"
}
```

## ✅ Answer

This block configures the Azure provider.

### Components:

| Component | Meaning |
|---|---|
| azurerm | Azure Resource Manager provider |
| features {} | Mandatory provider configuration |
| subscription_id | Azure subscription ID |

---

# 2️⃣1️⃣ What is a provider in Terraform?

## ✅ Answer

A provider is a plugin that allows Terraform to interact with cloud services.

---

## 📌 Examples

| Cloud | Provider |
|---|---|
| Azure | azurerm |
| AWS | aws |
| GCP | google |

---

# 2️⃣2️⃣ What is `required_providers` block?

## ✅ Answer

It defines:

- Provider source
- Provider version

---

## 📌 Example

```hcl
terraform {
  required_providers {
    azurerm = {
      source  = "hashicorp/azurerm"
      version = "4.73.0"
    }
  }
}
```

---

## 📌 Purpose

- Version consistency
- Compatibility control

---

# 2️⃣3️⃣ Difference between `terraform plan` and `terraform apply`

| terraform plan | terraform apply |
|---|---|
| Preview changes | Execute changes |
| Safe review step | Actual deployment |
| No infrastructure changes | Creates/modifies resources |

---

# 2️⃣4️⃣ Can Terraform work without a state file?

## ✅ Answer

No.

Terraform needs the state file to:

- Track infrastructure
- Compare changes
- Manage updates safely

Without state:

- Duplicate resources may be created
- Terraform loses infrastructure mapping

---

# 2️⃣5️⃣ Why is Terraform state management important?

## ✅ Answer

Terraform state management is critical because Terraform uses the state file to:

- Track infrastructure
- Detect configuration drift
- Manage dependencies
- Perform updates safely

---

# 📌 Production Best Practice

In production environments, state files are usually stored remotely.

Examples:

- Azure Storage Account
- AWS S3 Bucket
- Terraform Cloud

# 📂 Terraform File & Folder Creation Flow

| File / Folder | Created When | Purpose |
|---|---|---|
| `.terraform/` | After `terraform init` | Stores provider plugins |
| `.terraform.lock.hcl` | After `terraform init` | Locks provider versions |
| `terraform.tfstate` | After `terraform apply` | Stores infrastructure state |
| `terraform.tfstate.backup` | During next apply/change | Backup of previous state |
