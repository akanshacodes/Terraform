# 4. Terraform Variables

Variables allow Terraform configurations to become dynamic.

---

# Definition

```text
Variables are reusable placeholders used to store configurable values.
```

---

# Purpose

| Purpose | Benefit |
|---|---|
| Reusability | Avoid duplicate code |
| Flexibility | Dynamic infrastructure |
| Maintainability | Easier updates |
| Scalability | Multi-environment support |

---

# Example

```hcl
variable "rg_name" {
}

resource "azurerm_resource_group" "rg1" {
  name = var.rg_name
}
```

---

# 7. Variable Assignment Methods

Terraform supports multiple methods for assigning variable values.

---

# Method 1 — CLI Variables

Values passed during runtime.

---

# Example

```bash
terraform plan -var 'rg_name=rg-prod'
```

---

# Advantages

- Runtime flexibility
- Dynamic execution
- CI/CD compatibility

---

# Method 2 — Default Values

Default values inside variable blocks.

---

# Example

```hcl
variable "rg_name" {
  default = "rg-default"
}
```

---

# Benefits

- Simplified configuration
- Optional input support

---

# Method 3 — terraform.tfvars

Most common enterprise approach.

---

# Example

```hcl
rg_name     = "rg-prod"
rg_location = "westus"
```

---

# Purpose

Separates:
- configuration values
- infrastructure logic

---

# Method 4 — `.auto.tfvars`

Automatically loaded variable files.

---

# Benefits

- Automatic variable loading
- Reduced manual configuration

---

# Important Interview Point

```text
terraform.tfvars is commonly used to separate infrastructure configuration values from Terraform code.
```

---

| Feature | List |

- Subnet lists
- Region lists

---

# 16. Set Data Type

Collection containing unique values.

---

# Example

```hcl
toset(["rg1", "rg2", "rg3"])
```

---

# Characteristics

| Feature | Set |
|---|---|
| Ordered | No guarantee |
| Duplicates Allowed | No |
| Unique Values | Yes |

---

# Important Interview Point

```text
Sets automatically remove duplicate values.
```

---

# 17. Map Data Type

Key-value pair collection.

---

# Example

```hcl
regions = {
  rg1 = "westus"
  rg2 = "eastus"
}
```

---

# Characteristics

| Feature | Map |
|---|---|
| Key-Value Structure | Yes |
| Uses Curly Braces | Yes |
| Key Must Be String | Yes |

---

# Real Enterprise Use Cases

| Scenario | Example |
|---|---|
| Environment Mapping | dev → westus |
| RG Mapping | rg1 → eastus |
| VM Sizing | app1 → Standard_B2s |

---

# Important Interview Point

```text
Maps are commonly used in enterprise Terraform configurations for infrastructure relationships and environment mapping.
```

# 22. count vs for_each

Terraform supports two major looping mechanisms.

---

# Comparison

| Feature | count | for_each |
|---|---|
| Index-Based | Yes | No |
| Stable Resource Mapping | Limited | Better |
| Enterprise Preference | Less preferred | Industry standard |

---

# Important Interview Point

```text
for_each is preferred over count for stable resource tracking and scalable infrastructure design.
```

# 23. for_each Meta Argument

One of the most important Terraform concepts.

---

# Definition

```text
for_each dynamically creates multiple resource instances using collections.
```

---

# Example

```hcl
for_each = toset(["rg1", "rg2"])
```

---

# Supported Input Types

| Type | Supported |
|---|---|
| Set of Strings | Yes |
| Map | Yes |

---

# Important Interview Point

```text
for_each supports maps and sets of strings for dynamic resource creation.
```

# 24. for_each with Set

Used for creating multiple similar resources.

---

# Example

```hcl
resource "azurerm_resource_group" "rg" {

  for_each = toset(["rg1", "rg2"])

  name = each.value
}
```

---

# Internal Terraform Behavior

Terraform internally:
- iterates through collection
- creates multiple resources

---

# 25. for_each with Map

Production-grade infrastructure design pattern.

---

# Example

```hcl
resource "azurerm_resource_group" "rg" {

  for_each = {
    rg1 = "westus"
    rg2 = "eastus"
  }

  name     = each.key
  location = each.value
}
```

---

# Internal Logic

| Element | Meaning |
|---|---|
| each.key | Resource name |
| each.value | Resource value |

---

# Enterprise Use Cases

- Environment deployment
- Region mapping
- Dynamic VM creation
